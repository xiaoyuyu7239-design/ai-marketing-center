## 4. 异机回执与冻结窗口：尚未完成

生产可接受的备份证据不能只看 `manifest.json` 或目录修改时间。专用维护任务完成备份并把同一份 manifest 对应的对象同步到异机/对象存储后，必须取得真实版本回执，格式精确为：

```json
{
  "schemaVersion": 1,
  "provider": "真实存储提供方",
  "objectKey": "真实对象键",
  "versionId": "真实不可变版本号",
  "manifestSha256": "64位小写十六进制SHA-256",
  "syncedAt": "标准ISO时间"
}
```

然后在只写一次的证据目录运行：

```bash
pnpm ops:verify-backup -- \
  --backup /backups/backup-... \
  --evidence /release-evidence/backup-verification.json \
  --receipt /release-evidence/offsite-backup-receipt.json \
  --freeze-start 2026-07-16T12:30:00.000Z \
  --freeze-end 2026-07-16T12:45:00.000Z
```

工具要求冻结窗口覆盖备份 `createdAt` 且最长不超过 2 小时；异机 `syncedAt` 必须晚于备份；正式备份、证据文件和回执各自的 SHA-256 必须相互绑定。生产预检还会重新深度校验正式目录、SQLite/外键、证据新鲜度和时间顺序。

本轮 20:39 本地演练没有进入真实维护/只读窗口，也没有取得真实异机版本回执，因此不得为它伪造 `writesFrozen: true` 或站外回执。
