# 微信发布提醒 —— 服务号配置手册（运维侧）

这份文档给**部署/运营 SaaS 的人**看。配好之后，商家在「设置 → 发布提醒」里扫码绑定微信，到了他家客人最活跃的时段，就会收到"该发视频啦"的微信提醒。

商家侧完全无感知这些配置——他们只需要扫一个码。

## 一、需要准备什么

| 前置条件 | 说明 |
| --- | --- |
| 已认证的微信**服务号** | 订阅号/个人号没有模板消息权限，必须是企业认证服务号 |
| 模板消息能力 | 服务号后台「广告与服务 → 模板消息」开通，行业选电商/生活服务类 |
| 公网可访问的部署地址 | 微信回调要能打到你的服务器（`/api/wechat/callback`） |

## 二、服务号后台配置（mp.weixin.qq.com）

1. **拿凭证**：「设置与开发 → 基本配置」里拿 AppID 和 AppSecret（AppSecret 只显示一次，存好）。把服务器公网 IP 加入「IP 白名单」，否则拿不到 access_token。
2. **配回调**：同页「服务器配置（启用）」：
   - URL 填 `https://你的域名/api/wechat/callback`
   - Token 自己起一个随机串（下面环境变量要用同一个）
   - 消息加解密方式选**明文模式**
   - 点提交，微信会立刻发一次 GET 验证，服务已启动且环境变量配好才能通过
3. **建模板**：「模板消息 → 从模板库选用」，选一个含四个字段的通用提醒模板。默认代码按经典字段发送：`first`（标题）、`keyword1`（正文）、`keyword2`（时间）、`remark`（尾注）。如果你选的模板字段名不同（如类目模板的 `thing2`/`time3`），用 `WECHAT_TEMPLATE_FIELDS` 环境变量映射，见下表。

## 三、环境变量

```bash
# 必填四件套 —— 缺任何一个，微信提醒整体静默关闭（站内时段提示不受影响）
WECHAT_APP_ID=wx1234567890abcdef
WECHAT_APP_SECRET=你的AppSecret
WECHAT_CALLBACK_TOKEN=与服务号后台"服务器配置"里一致的Token
WECHAT_TEMPLATE_ID=模板消息id

# 可选
APP_BASE_URL=https://你的域名        # 提醒消息点开跳转到"今日待发布"，不填则消息不带链接
WECHAT_TEMPLATE_FIELDS='{"title":"thing2","body":"thing3","date":"time4","remark":"thing5"}'  # 模板字段名映射，默认经典 first/keyword1/keyword2/remark
WECHAT_DRY_RUN=1                    # 干跑模式：不真发微信只打日志，本地联调用
CLIPFORGE_DISABLE_SCHEDULER=1       # 彻底关掉提醒定时器（如多实例部署时只留一台跑）
TZ=Asia/Shanghai                    # 重要：时段判定用服务器本地时钟，海外服务器必须设
```

## 四、它是怎么跑起来的

- **定时器**：Next.js 启动时（`src/instrumentation.ts`）拉起每分钟一跳的调度器，检查每个"开了提醒且绑了微信"的商家。
- **时段**：起步用行业经验时段表（按品类 + 实体店/电商区分）；商家回流的发布数据攒够 8 条后，自动换成按他家自己数据校准的时段。
- **防打扰**：每商家每个时段每天最多提醒一次（`reminder_logs` 表去重）；当天目标条数已发完就不再提醒；一天最多 3 个时段。
- **绑定**：商家在设置页点"扫码绑定"→ 后端生成 10 分钟有效的带参二维码 → 扫码关注后微信回调带 `bind:商家id` → 自动落绑定。一个商家可以绑多个微信（老板 + 店员），取关自动解绑。

## 五、排查"没收到提醒"

按顺序查：

1. 服务器日志有没有"获取微信 access_token 失败"→ 检查 AppSecret 和 IP 白名单。
2. `reminder_logs` 表有没有该商家当天的记录：
   - 没有 → 商家没开开关 / 没绑微信 / 当前不在时段窗口；
   - `status=skipped` → 当天目标已完成，正常；
   - `status=failed` → 看 `detail` 里的微信错误码（43004=用户未关注，40003=openid 不对，40037=模板 id 不对）。
3. 商家是不是取关了服务号（取关会自动删绑定）。
