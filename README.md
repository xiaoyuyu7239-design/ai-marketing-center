# AI Marketing Center

AI Marketing Center 是把 [ClipForge](https://github.com/xixihhhh/clipforge)（原名"带货剪手"，开源短视频创作引擎）转型为面向中小商家的 **Web 端订阅制 SaaS** 的在建项目：目标是把商品、卖点、脚本、素材、视频、文案和发布提醒组织成一条老板能持续执行的营销内容生产流程。

核心心智（目标态，见下方"当前实现状态"了解哪些已经做到）：老板不用学短视频运营，只需要上传产品、做选择题、确认发布；AI 负责分析、生成、提醒和辅助发布。

> 本 README 只描述本仓库实际存在、可复现的能力，以及明确标注为"规划中"的目标。不在"已实现"里出现的内容，代码里都还没有。

## 项目定位

中小商家做短视频营销时，真正困难的往往不是单点剪辑，而是整套运营动作都很重：不知道发什么、不会写标题文案、拍摄剪辑成本高、网感不足、发布节奏难坚持，还容易错过适合发布的时间。

AI Marketing Center 的目标不是做一个孤立的视频生成器，而是把短视频营销拆成低门槛、可复用、可管理的工作流：

```text
商家建档 → 上传产品 → AI 分析卖点 → AI 推荐内容方向 → 老板点选
→ AI 批量生成营销视频包 → 存入待发布库 → 发布提醒 → 老板勾选 → 手动发布
```

## 目标用户

第一阶段面向有产品、有素材、想持续发短视频但缺少专业运营能力的中小商家。

优先覆盖 5 个模板基础清晰、短视频营销需求强的品类：

| 品类 | 典型内容方向 |
| --- | --- |
| 美妆护肤 | 痛点种草、成分亮点、使用前后、场景安利 |
| 食品零食 | 口感展示、囤货推荐、场景分享、对比测评 |
| 家居日用 | 使用场景、痛点解决、功能展示、清洁收纳 |
| 服饰鞋包 | 上身效果、穿搭组合、风格推荐、细节质感 |
| 数码 3C | 功能卖点、参数解释、场景演示、对比测评 |

产品形态只做 **响应式 Web 端 + 内部运营后台**，不做商家小程序端（已评估过技术栈和微信审核成本，决定不投入）。

这是转型工作里最重要的一张表，逐条对应最初立项方案的产品闭环，如实标注现状，会随每个阶段上线同步更新。

| 环节 | 状态 | 说明 |
| --- | --- | --- |
| 商家账号 / 多租户 | ✅ 已实现 | 邮箱密码注册/登录（`/api/auth/*`），所有项目数据按商家隔离；桌面版/CLI 走单用户模式（`CLIPFORGE_SINGLE_USER=1`）免登录 |
| 商家建档 | ✅ 已实现 | 设置中心"商家信息"填品类/地区/目标客户/价格带/主投平台，生成脚本时自动作为默认值注入 |
| 商品图/商品链接导入 | ✅ 已实现 | 支持传图，也支持粘贴商品链接自动抓取标题/价格/图（`后端/core/stock/product-ingest.ts`） |
| AI 卖点分析 | ✅ 已实现 | 分析商品图提炼卖点/人群/痛点，写入 `productAnalysis`（`后端/script-engine/generator.ts`） |
| 内容方向"点选" | ✅ 已实现 | 工作台"更多设置"里选内容方向：智能推荐/痛点种草/场景安利/对比测评/剧情故事 |
| 视频生成（脚本/素材/合成/字幕/BGM/标题/文案/封面/话题标签） | ✅ 已实现 | 核心生成流水线质量扎实，是新 SaaS 复用的底层能力 |
| 待发布库 | ✅ 已实现 | 认可入库/发布状态服务端持久化（`publish_records` 表），换设备不丢；LLM/规则双路择优今日待发布 |
| 发布提醒 | 🚧 部分实现 | 按品类的黄金时段建议（`后端/core/publish/golden-time.ts`）+ 应用内提醒已接入；微信/短信等站外触达渠道未选型 |
| 跳转平台发布 | ❌ 不做真跳转（产品决策） | 深链+自动预填通常只对认证机构开放；现状是一键复制完整文案包 + 多平台比例导出下载 + 标记已发布，商家手动打开 App 发布 |
| 生成配额计量 | ✅ 已实现 | 每次生成按商家记账（`generation_usage`），套餐额度用尽返回 402；试用套餐 20 次/月 |
| 套餐 / 订阅计费 | 🚧 部分实现 | 套餐、配额拦截、运营后台调套餐/赠额度均已有；仍无支付通道/商家自助升级（待支付选型） |
| 托管 AI 模型 | ✅ 已实现 | 模型凭证由服务端环境变量/运营后台统一配置（`服务器/admin/agents/`），商家无需自带 Key |
| 商家小程序端 | ❌ 不在计划内（产品决策） | 只做 Web 端 |
| 内部运营后台 | ✅ 已实现 | `/admin` 新增商家管理（套餐调整/赠送额度/本月用量）与内容审核队列（看片/通过/驳回，驳回即从商家待发布消失），原有 AI 调试面板（prompt/模型评测/生成记录）保留 |

## 视频生成流水线（已实现）

视频生成模块是一条标准内容生产流水线，而不是单次按钮触发的孤立能力：

```text
产品图 → 卖点分析 → 脚本分镜 → 素材补齐 → 混剪成片 → 文案封面 → 入库待发
```

当前重点支持两类视频模式：

| 模式 | 适合场景 | 生成策略 |
| --- | --- | --- |
| 图文混剪 | 快消品、食品零食、家居日用等低门槛快速出片场景 | 产品图 + 卖点字幕 + 文字卡片 + 转场 + BGM + 通用场景素材 |
| 产品特写 | 质感型商品、高客单价商品、需要突出外观细节的商品 | 基于产品图做推进、横移、光影、卖点贴片，尽量保持产品主体稳定 |

此外还有发布前限流自检（广告法风险词/钩子强度/时长/字幕/CTA/AIGC 标识）和效果回流（按脚本风格聚合转化数据）两项立项方案未设想、但已经落地的能力。

## 系统架构

| 层级 | 组成 |
| --- | --- |
| 用户层 | 商家 Web 端、内部运营后台（不含小程序）；桌面版/CLI 以单用户模式复用同一套服务 |
| 应用层 | 内容生产 workflow、Agent 角色分工、公共工具能力 |
| 模型层 | 平台托管的 LLM、图片编辑模型、图生视频模型、TTS，按商家配额计量 |
| 基础层 | 品类模板库、钩子库、SEO 策略、合规词库、商家账号/建档画像/套餐/待发布库数据 |

转型分阶段推进：Phase 0（文档诚实化 + 修复 AI 分析默认被跳过）→ Phase 1（商家账号体系 + 托管模型计量 + 配额拦截）→ Phase 2（商家建档 + 内容方向点选 + 待发布库落库 + 黄金时间提醒 + 手动发布打磨）→ Phase 3（运营后台：商家管理/内容审核）——四个阶段均已完成。商业化剩余项（支付通道/提醒触达渠道/定价）见「后续方向」。

## 模型与工具策略

| 类型 | 用途 | 当前 Key 来源 |
| --- | --- | --- |
| 文本模型 | 脚本、卖点、标题、发布文案、SEO 和提示词生成 | 平台托管（服务端环境变量 / `/admin` 后台策略），商家无感 |
| 图片模型 | 商品图处理、背景生成、商品主体保真编辑 | 同上 |
| 视频模型 | 图生视频、商品动态镜头和画面动效 | 同上 |

| 合规工具 | 风险词检查、平台适配、AIGC 标识和发布前自检 | 纯本地规则，无需 Key |

## 本地运行

运行环境：

- Node.js >= 20
- pnpm >= 10

```bash
pnpm install   # 安装依赖
pnpm dev       # 启动开发服务，默认 http://localhost:3000
```

接口默认要求商家登录（首页可注册）。本机单人使用（含 CLI/MCP 驱动）可开启单用户模式跳过登录：

```bash
CLIPFORGE_SINGLE_USER=1 pnpm dev   # 桌面版已内置开启；多商家部署严禁开启
```

项目首页会进入 `/start`，核心创作工作台入口为 `/project/agent`。

## 主要目录

业务代码实际分布在三个中文目录里（`tsconfig.json` 里的 `@frontend`/`@backend`/`@server` 路径别名指向它们），`src/app` 下的页面文件大多只是转发壳（如 `export { default } from "@frontend/创作工作台/page"`）：

```text
前端/                页面与前端逻辑（@frontend），按功能模块分目录，如 创作工作台/、导出发布/、后台管理/
后端/                业务引擎（@backend）：script-engine 脚本生成、video-composer 合成、providers 模型适配、core/publish 发布文案、db 数据库 schema
服务器/              服务端能力（@server）：目前主要是内部 admin（agent/prompt/模型评测）
src/app/             Next.js 路由入口，多数页面转发到 前端/ 对应模块
src/app/api/         服务端接口
src/lib/__tests__/   测试代码
public/              静态资源与展示素材
drizzle/             数据库迁移
electron/            桌面应用入口与打包配置
mcp/                 Agent / MCP 接入能力（ClipForge 阶段遗留，供 AI Agent/CLI 调用）
```
