# AI Marketing Center

AI Marketing Center 是一个面向中小商家的 AI 短视频营销工作台，用来把商品、素材、卖点、脚本、分镜、视频、标题、封面和发布提醒组织成一条可持续执行的营销内容生产流程。

核心心智：老板不用学短视频运营，只需要上传产品、做选择题、确认发布；AI 负责分析、生成、提醒和辅助发布。

## 项目定位

中小商家做短视频营销时，真正困难的往往不是单点剪辑，而是整套运营动作都很重：不知道发什么、不会写标题文案、拍摄剪辑成本高、网感不足、发布节奏难坚持，还容易错过适合发布的时间。

AI Marketing Center 的目标不是做一个孤立的视频生成器，而是把短视频营销拆成低门槛、可复用、可管理的工作流：

```text
商家建档
→ 上传产品
→ AI 分析卖点
→ AI 推荐内容方向
→ 老板点选
→ AI 批量生成营销视频包
→ 存入待发布库
→ 黄金时间提醒
→ 老板勾选
→ 跳转平台手动发布
```

## 目标用户

第一阶段面向有产品、有素材、想持续发短视频但缺少专业运营能力的中小商家。

优先覆盖 5 个模板基础清晰、短视频营销需求强的品类：

| 品类 | 典型内容方向 |
| --- | --- |
| 美妆护肤 | 痛点种草、成分亮点、使用前后、场景安利 |
| 食品零食 | 口感展示、囤货推荐、场景分享、对比测评 |
| 家居日用 | 使用场景、痛点解决、功能展示、清洁收纳 |
| 服饰鞋包 | 上身效果、穿搭组合、风格推荐、细节质感 |
| 数码 3C | 功能卖点、参数解释、场景演示、对比测评 |

## 核心产品闭环

| 阶段 | 商家动作 | AI / 系统产出 |
| --- | --- | --- |
| 商家建档 | 填写品类、地区、价格带、目标客户、品牌信息 | 建立商家画像，作为后续推荐依据 |
| 上传产品 | 上传产品图、包装图、短视频或门店素材 | 识别产品外观、品类、场景和可用素材 |
| 卖点分析 | 查看分析结果 | 提炼卖点、人群、痛点、差异化和表达方式 |
| 方向推荐 | 点选内容方向、卖点或风格 | 生成个性化内容 brief |
| 视频生成 | 等待生成，可选择是否加入配音 | 生成 9:16 视频、字幕、BGM、封面、标题、文案和标签 |
| 待发布库 | 查看、筛选、勾选内容 | 按产品、平台、方向、时间、审核状态管理内容 |
| 发布提醒 | 接收微信服务通知或闹铃提醒 | 在适合时段提醒发布，降低遗忘和拖延 |
| 辅助发布 | 跳转平台并手动确认发布 | 尽量带入视频、标题、文案、封面等发布材料 |

## 视频生成流水线

视频生成模块是一条标准内容生产流水线，而不是单次按钮触发的孤立能力。

```text
产品图
→ 卖点分析
→ 内容方向
→ 脚本分镜
→ 素材补齐
→ 混剪成片
→ 文案封面
→ 入库待发
```

当前 MVP 重点支持两类视频模式：

| 模式 | 适合场景 | 生成策略 |
| --- | --- | --- |
| 图文混剪 | 快消品、食品零食、家居日用等低门槛快速出片场景 | 产品图 + 卖点字幕 + 文字卡片 + 转场 + BGM + 通用场景素材 |
| 产品特写 | 质感型商品、高客单价商品、需要突出外观细节的商品 | 基于产品图做推进、横移、光影、卖点贴片，尽量保持产品主体稳定 |

## 系统架构

### 六层产品架构

| 层级 | 关注点 | 说明 |
| --- | --- | --- |
| 市场层 | 服务谁、为什么做 | 聚焦短视频带货营销，服务 5 大品类中小商家 |
| 商业层 | 价值主张与模式 | 提供持续发视频的能力，支持订阅、套餐和次数机制 |
| 用户层 | 老板怎么用 | 把创作变成上传、点选、确认三个轻动作 |
| 技术层 | 怎么实现 | 用确定性 workflow 串联卖点分析、脚本、素材、合成和发布辅助 |
| 模型层 | 调用哪些 AI | 文本、图片、视频、音频模型按环节组合，强模型用于关键节点 |
| 基础层 | 靠什么越用越准 | 模板库、素材库、商家画像、生成记录、待发布库和后续数据回流 |

| 应用层 | 内容生产 workflow、Agent 角色分工、公共工具能力 |

## Agent 与 Workflow 设计

当前项目以确定性内容生产流水线为核心，把一个短视频运营团队的分工映射成可编排的系统节点。

| 角色 | 职责 |
| --- | --- |
| 营销总监 | 编排流程、把控内容方向和整体节奏 |
| 卖点分析师 | 从商品信息和素材中提炼卖点、人群、痛点和差异化 |
| 选题策划师 | 根据品类、平台和目标人群推荐内容方向 |
| 编剧 / 分镜 | 生成短视频脚本、镜头结构、字幕节奏和画面提示 |
| 文案 / SEO | 生成标题、发布文案、话题标签和封面大字 |
| 视觉设计 | 负责商品图处理、画面风格、背景和素材补齐 |
| 配音 / 音乐 | 处理 TTS、BGM、字幕节奏和声音策略 |
| 合规审查 | 检查广告法风险词、平台风险点和商品保真问题 |
| 发布运营 | 管理待发布库、推荐发布时间、提醒和跳转发布 |

## 三端产品结构

| 入口 | 使用对象 | 核心功能 |
| --- | --- | --- |
| 商家小程序端 | 老板、店主、商家员工 | 建档、上传产品、选择方向、查看生成进度、接收提醒、跳转发布 |
| 商家 Web 端 | 有电脑端管理需求的商家 | 产品库、素材库、视频库、生成记录、套餐余额、批量下载 |
| 内部运营后台 | 运营、客服、管理人员 | 商家管理、套餐次数管理、任务监控、视频审核、模板管理、数据查看 |

## 当前 MVP 边界

当前版本优先跑通从商品到待发布内容的核心闭环。

已纳入 MVP 的能力：

- 商家建档与产品上传
- AI 卖点分析与内容方向推荐
- 老板点选方向后生成视频包
- 9:16 竖屏营销视频生成
- 字幕、BGM、标题、文案、封面和话题标签输出
- 待发布库管理
- 微信服务通知 / 闹铃提醒
- 跳转抖音、小红书、视频号后由老板手动确认发布
- 内部后台管理商家、套餐、任务和审核

暂不纳入 MVP 的能力：

- 真人出镜
- 数字人
- 自动发布
- 账号数据回流
- 小红书 3:4 专版
- 复杂场景演示
- 长视频切片

## 本地运行

运行环境：

- Node.js >= 20
- pnpm >= 10

安装依赖：

```bash
pnpm install
```

启动开发服务：

```bash
pnpm dev
```

默认访问地址：

```text
http://localhost:3000
```

项目首页会进入：

```text
/start
```

核心创作工作台入口：

```text
/project/agent
```
